## 🛠️ How to Contribute

We encourage contributions from developers of all skill levels. To get started:

1.  **Fork** the repository.
2.  **Clone** your fork locally and create a new feature branch: `git checkout -b feature/your-feature-name`
3.  **Code** your addition. Remember "The Coding Creed": *Keep it simple.*
4.  **Document** your code. A repository with a solid DOCS page is worth its weight in gold. Ensure instructions are clear.
5.  **Commit** your changes: `git commit -m "Add short description"`
6.  **Push** to your branch: `git push origin feature/your-feature-name`
7.  **Submit** a Pull Request (PR) for review.

*Note: All code submitted here will be subject to our community review protocols. Please ensure you are not posting malicious or unverified scripts directly into the main branch.*

## ⚠️ Security & Guidelines

*   **The Protocol**: All code must be safe for public consumption. Code that is deemed experimental or potentially harmful should be verified in the server's Private Channels (The Review Chamber) before submission.
*   **Open Source Sharing**: By contributing, you agree that your code will be freely shared for the community to learn from and use under the [MIT License](LICENSE) (or other specified open-source license).
*   **Zero Tolerance**: Any intentional submission of malware, phishing scripts, or deceptive code will result in an immediate strike, a ban review, and the destruction of the submitted data, as outlined in our server rules.
